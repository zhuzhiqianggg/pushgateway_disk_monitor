# 硬盘寿命查看与计算指南

## 一、概述

本文档详细说明如何通过 S.M.A.R.T. 数据查看和计算 SSD 硬盘寿命，包括 NVMe 和 SATA 两种接口的不同指标。

---

## 二、NVMe 硬盘寿命指标

### 2.1 核心指标

| 指标名称 | smartctl 输出字段 | 含义 |
|---------|------------------|------|
| Percentage Used | `Percentage Used:` | 已使用寿命百分比 (0-100%) |
| Available Spare | `Available Spare:` | 剩余备用块百分比 (0-100%) |
| Available Spare Threshold | `Available Spare Threshold:` | 备用块告警阈值 |
| Data Units Written | `Data Units Written:` | 总数据写入量 (512-byte blocks) |
| Power On Hours | `Power On Hours:` | 通电时间 (小时) |

### 2.2 查看命令

```bash
# 查看 NVMe 硬盘详细信息
smartctl -A /dev/nvme0n1

# 示例输出
SMART/Health Information (NVMe Log 0x02)
Critical Warning:                   0x00
Temperature:                        40 Celsius
Available Spare:                    100%
Available Spare Threshold:          10%
Percentage Used:                    4%
Data Units Written:                 174,851,279 [89.5 TB]
Power On Hours:                     15,322
```

### 2.3 寿命计算方法

```
已使用百分比 = Percentage Used (直接读取)
剩余百分比 = 100 - Percentage Used
预计剩余寿命(小时) = (100 - Percentage Used) * Power_On_Hours / Percentage Used
设计总寿命(小时) = Power_On_Hours * 100 / Percentage Used
```

### 2.4 已知问题

**INSPUR 品牌 NVMe 固件 Bug**：
- 型号：INSPUR-NS8500G2U384、INSPUR-NS6510G1U384 等
- 问题：`Percentage Used` 始终显示 0%，即使硬盘已使用多年
- 验证方法：对比 `Power On Hours` 和 `Data Units Written`，如果有大量写入但 Percentage Used = 0，说明存在固件 bug
- 建议：联系厂商获取固件更新，或通过 `Data Units Written` / 容量 推算 DW (Disk Writes) 次数

**其他品牌**：
- 大部分 NVMe 硬盘（Intel、Samsung、Micron）的 `Percentage Used` 是准确的

## 三、SATA SSD 硬盘寿命指标

### 3.1 核心指标

| 指标名称 | SMART ID | 含义 |
|---------|----------|------|
| Wear_Leveling_Count | 177 | 磨损均衡计数 |
| Media_Wearout_Indicator | 233 | 介质磨损指示器（Intel 专用） |
| Total_LBAs_Written | 241 | 总 LBA 写入量 (512-byte blocks) |
| Power_On_Hours | 9 | 通电时间 (小时) |

### 3.2 查看命令

```bash
# 查看 SATA SSD 详细 SMART 数据
smartctl -A /dev/sda

# 示例输出
ID# ATTRIBUTE_NAME          FLAG     VALUE WORST THRESH TYPE      UPDATED  WHEN_FAILED RAW_VALUE
  5 Reallocated_Sector_Ct   0x0033   100   100   010    Pre-fail  Always       -       0
  9 Power_On_Hours          0x0032   095   095   000    Old_age   Always       -       20293
177 Wear_Leveling_Count     0x0013   099   099   000    Pre-fail  Always       -       9
241 Total_LBAs_Written      0x0032   099   099   000    Old_age   Always       -       118312627673
```

### 3.3 SMART 字段说明

| 字段 | 说明 |
|------|------|
| VALUE | 标准化值，通常是剩余寿命百分比 (如 094 = 剩余 94%) |
| WORST | 历史最差值 |
| THRESH | 告警阈值，低于此值表示硬盘即将失效 |
| RAW_VALUE | 原始值，不同品牌含义不同 |

### 3.4 寿命计算方法

**方法一：Wear_Leveling_Count VALUE 字段（推荐）**
```
剩余百分比 = VALUE 字段 (如 094 = 94%)
已使用百分比 = 100 - VALUE
```

**方法二：Media_Wearout_Indicator VALUE 字段（Intel 专用）**
```
剩余百分比 = VALUE 字段
已使用百分比 = 100 - VALUE
```

**方法三：Wear_Leveling_Count RAW_VALUE（部分品牌）**
```
已使用百分比 = RAW_VALUE (如 Samsung 870 QVO)
```

### 3.5 不同品牌的差异

| 品牌/型号 | Wear_Leveling_Count VALUE | Wear_Leveling_Count RAW_VALUE | 推荐算法 |
|----------|--------------------------|------------------------------|----------|
| Samsung 870 QVO | 剩余百分比 (如 094 = 94%剩余) | 已使用百分比 (如 9 = 9%已用) | VALUE 或 RAW_VALUE |
| Samsung 870 EVO | **不确定，可能不是剩余百分比** | **不确定，可能是磨损次数** | 需要验证 |
| Samsung 860 EVO | 剩余百分比 | 不确定 | VALUE |
| Intel SSD | 使用 Media_Wearout_Indicator | - | MWI VALUE |
| Micron/Crucial | 剩余百分比 | 不确定 | VALUE |
| INAGILE | 剩余百分比 | 不确定 | VALUE |

## 四、常见问题排查

### 4.1 如何验证寿命百分比是否准确？

**步骤 1：查看 SMART 原始数据**
```bash
smartctl -A /dev/sda
smartctl -A /dev/nvme0n1
```

**步骤 2：对比多块同型号硬盘**
- 通电时间相近的硬盘，寿命百分比应该相近
- 如果某块盘明显偏高/偏低，说明算法可能不准确

**步骤 3：检查写入量**
```bash
# SATA SSD
smartctl -A /dev/sda | grep Total_LBAs_Written

# NVMe
smartctl -A /dev/nvme0n1 | grep "Data Units Written"
```
- 写入量大的盘，寿命百分比应该更高
- 如果写入量很小但百分比很高，说明算法有问题

**步骤 4：查看厂商规格**
- 搜索硬盘型号的 datasheet
- 查看 TBW (Terabytes Written) 额定值
- 对比实际写入量与 TBW 的比值

### 4.2 NVMe Percentage Used = 0 的排查

```bash
# 1. 确认是否固件 bug
smartctl -A /dev/nvme0n1 | grep -E "(Percentage Used|Power On Hours|Data Units Written)"

# 2. 如果 Power On Hours > 100 且 Data Units Written 很大，但 Percentage Used = 0
#    说明存在固件 bug

# 3. 临时解决方案：通过 DW 比率推算
# DW = Data Units Written (TB) / 硬盘容量 (TB)
# 已使用百分比 ≈ DW / 3000 * 100  (假设 3000 DW = 100% 寿命)
```

### 4.3 SATA SSD 寿命百分比异常的排查

```bash
# 1. 查看 Wear_Leveling_Count 的 VALUE 和 RAW_VALUE
smartctl -A /dev/sda | grep -i "Wear_Leveling_Count"

# 2. 如果 VALUE 接近 100 但 RAW_VALUE 很小
#    说明 VALUE = 剩余百分比，RAW_VALUE = 磨损次数

# 3. 如果 VALUE 接近 0 但 RAW_VALUE 很大
#    说明 VALUE = 已使用百分比 (少见)

# 4. 对比同型号其他硬盘的 SMART 数据
#    确认 VALUE 和 RAW_VALUE 的含义
```

## 五、监控脚本中的实现

### 5.1 当前逻辑 (disk_health_monitor.sh)

**NVMe:**
```bash
# 直接读取 Percentage Used
value=$(smartctl -A "$device" | grep 'Percentage Used:' | awk '{print $1}' | tr -d '%,')
# 如果获取不到，返回 -1
```

**SATA SSD:**
```bash
# 方法1: 查找 Percent_Lifetime_Used 或 Percentage_Used (罕见)
# 方法2: Wear_Leveling_Count VALUE → 100 - VALUE (推荐)
# 方法3: Media_Wearout_Indicator VALUE → 100 - VALUE (Intel)
# 方法4: Wear_Leveling_Count RAW_VALUE (Samsung QVO 等)
```

### 5.2 PushGateway 指标

| 指标名 | 含义 | 单位 |
|-------|------|------|
| `smart_percentage_used` | 已使用寿命百分比 | 0-100, -1=不可获取 |
| `smart_remaining_life_hours` | 预计剩余寿命 | 小时 |
| `smart_disk_max_life_hours` | 推算设计总寿命 | 小时 |
| `smart_power_on_hours` | 通电时间 | 小时 |
| `smart_total_lbas_written` | 总写入量 (SATA) | 512-byte blocks |
| `smart_data_written_blocks` | 总写入量 (NVMe) | 512-byte blocks |

## 八、排查步骤模板

当你发现某块硬盘的寿命百分比异常时，按以下步骤排查：

```bash
# 1. 查看完整 SMART 数据
smartctl -A /dev/sdX

# 2. 查看硬盘信息
smartctl -i /dev/sdX

# 3. 记录以下关键值：
#    - Power_On_Hours (RAW_VALUE)
#    - Wear_Leveling_Count (VALUE 和 RAW_VALUE)
#    - Total_LBAs_Written (RAW_VALUE)
#    - Reallocated_Sector_Ct (RAW_VALUE)

# 4. 对比同型号其他硬盘
#    找几块同型号、通电时间相近的盘，对比 SMART 数据

# 5. 计算理论 TBW
#    实际写入 TB = Total_LBAs_Written * 512 / 1024^4
#    厂商 TBW = 搜索型号 datasheet
#    理论寿命% = 实际写入 TB / 厂商 TBW * 100

# 6. 对比理论值和脚本计算值
#    如果差异很大，说明脚本算法需要调整
```
